# 商情管理系统本地全栈设计

## 背景

依据项目根目录中的《商情管理系统_需求文档_PC端单角色精简版_副本.docx》，实现一个可本地部署的 PC 端商情管理系统。系统面向单一角色“客户经理”，覆盖客户走访、商情跟踪、客户管理和数据看板四个模块。

## 技术方案

采用 React/Vite 前端、Node.js/Express 后端、SQLite 本地数据库。

该方案保留前后端分离结构，支持真实 REST API、数据持久化、表单校验和状态流转；同时部署成本低，适合本地演示与后续迁移到 MySQL 或其他服务端数据库。

## 功能范围

### 商情跟踪

- 商情列表：统计卡、阶段步骤条筛选、关键字搜索、等级/类型/行业/时间筛选、分页表格。
- 商情详情：头部信息、阶段进度、业务信息、费用信息、联系人信息、客户走访时间线。
- 商情录入与编辑：客户联想、商机信息、费用信息、联系人信息，预估利润率实时计算。
- 状态流转：支持 10 商情发现、20 跟进中、30 商务洽谈、40 已转化、90 已终止。
- 操作规则：进行中阶段可推进、回退、终止；已终止可重新激活；已转化只读。

### 客户走访

- 走访列表：按客户聚合统计，展示最近走访对象、最近走访时间、累计走访次数和状态。
- 新增走访：支持从商情详情带入关联商情和客户，也支持列表入口新增。
- 走访记录：写入后回显到商情详情的走访时间线。
- 附件：支持最多 3 张现场照片的本地预览与保存。

### 客户管理

- 客户列表：客户名称、编号、行业、省份、联系人、关联商情数、累计走访次数。
- 客户详情：基本信息、联系人信息、关联商情、走访记录。
- 新增与编辑：维护客户基础档案和联系人。

### 数据看板

- 6 指标卡：总商情数、进行中商情数、已转化商情数、总商情金额、产出商情客户数、商情转化率。
- 转化漏斗：商情发现、跟进中、商务洽谈、已转化四级。
- 走访趋势：近数月走访次数与覆盖客户数。
- 业绩盘：按商机类型统计商情数、已转化数和转化率。

## 前端设计

前端采用 PC 管理后台布局：

- 左侧导航包含数据看板、商情跟踪、客户走访、客户管理。
- 主内容区使用 1440px 设计思路，兼容 1280px 以上桌面浏览器。
- 视觉主色为 `#2F88FF`，配合橙色、绿色、紫色和红色标签表达业务状态。
- 常用控件包含统计卡、表格、筛选栏、步骤条、分组表单、详情描述列表、时间线和图表。
- 图表使用轻量图表库渲染漏斗、柱状趋势和类型统计。

## 后端设计

后端提供 REST API：

- `/api/dashboard`：看板聚合数据。
- `/api/opportunities`：商情列表、新增、详情、编辑。
- `/api/opportunities/:id/status`：商情推进、回退、终止、重新激活。
- `/api/visits`：走访列表、新增、详情。
- `/api/customers`：客户列表、新增、详情、编辑。
- `/api/dictionaries`：阶段、等级、类型、行业等字典。

接口统一返回 `{ success, data, message }`，表单错误返回明确字段提示。

## 数据模型

SQLite 保留核心表：

- `customer`：客户档案。
- `opp`：商情主表。
- `opp_invest`：商情费用表。
- `opp_contact`：商情联系人表。
- `visit`：客户走访表。
- `visit_photo`：走访附件表。

服务启动时自动创建表并写入一批示例数据，便于本地打开后直接体验完整流程。

## 校验与错误处理

- 前端校验必填项、金额、手机号、日期和附件数量。
- 后端重复校验关键规则，避免绕过前端导致脏数据。
- 提交、删除、阶段流转、终止商情等操作提供确认或结果提示。
- 空列表、加载中和接口异常提供明确状态。

## 本地部署

项目启动后提供：

- 后端服务端口：`3001`。
- 前端开发服务端口：`5173`。
- 前端通过 Vite 代理访问 `/api`。
- 项目初始化为本地 Git 仓库，可连接到 GitHub 远程仓库。

最终交付应包含安装说明、启动命令和本地访问地址。

## 测试与验收

- 后端至少覆盖状态流转、利润率计算、列表查询、客户/走访创建等关键路径。
- 前端通过构建检查验证页面可编译。
- 启动本地服务后，用浏览器检查主页面、列表、详情、表单和看板正常渲染。
